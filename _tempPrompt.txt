Fix Graph RAG Visualization System - Event Not Reaching Dashboard

## Problem
Created a real-time visualization system for Graph RAG queries. Queries execute successfully and return results, but visualization events aren't reaching the browser dashboard. The server runs but emits no logs when queries execute.

## Architecture
- Backend: Python instrumentation system that emits events during query execution
- Server: FastAPI WebSocket server on port 8001 that broadcasts events
- Frontend: Browser dashboard connecting via WebSocket to receive events
- Flow: Query 창혛혪 QueryTracer.emit() 창혛혪 VisualizationHub 창혛혪 visualization_server 창혛혪 WebSocket 창혛혪 Dashboard

## What Works
창혵혫 Queries execute and return correct results
창혵혫 FastAPI server runs on port 8001
창혵혫 Dashboard connects to WebSocket (shows "챨혺혺짖 Connected")
창혵혫 Manual hub.emit() test events reach the manager queue

## What's Broken
창혵혰 No log output when running queries through query_instrumented functions
창혵혰 Events don't appear on dashboard when running: `from senpai.graph.query_instrumented import *; reps_who_win_viz(category="찾혗쨉찾혘쩌찾혘혨찾혘쩌")`
창혵혰 No "Event queued:" logs appear in server output
창혵혰 Dashboard remains blank with 0 events

## Files Involved
1. `senpai/graph/visualization.py` - QueryTracer class, VisualizationHub, create_query_tracer()
2. `senpai/graph/query_instrumented.py` - reps_who_win_viz(), account_graph_viz(), etc.
3. `senpai/api/visualization_server.py` - FastAPI server with WebSocket broadcast

## Test Results
- When importing visualization_server, hub.subscribers goes from 0 to 1 창혵혫
- Manual hub.emit() creates event history and queues event 창혵혫
- But reps_who_win_viz() query produces NO events in hub.event_history
